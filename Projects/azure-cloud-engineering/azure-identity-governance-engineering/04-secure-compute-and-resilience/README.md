# ⚖️ Project 04: Secure Compute & Resilience (In Progress)

## 🎯 Objective

Design a resilient and scalable compute environment using Azure, focusing on **high availability, controlled scaling, and operational monitoring**.

---

## 📌 Current Status

This project outlines the intended design for a **secure and highly available compute architecture**, including:

- Virtual Machine Scale Sets (VMSS)
- Load Balancing for traffic distribution
- Autoscaling based on demand
- Monitoring and alerting using Azure Monitor

Implementation is currently in progress.

---

## 🧠 Intended Design

The architecture will focus on:

- Eliminating single points of failure  
- Automating deployment and configuration  
- Enforcing consistent compute environments  
- Integrating monitoring and alerting for operational visibility  

---

## 🔐 Security & IAM Considerations

- Access to compute resources governed via RBAC  
- Integration with network security controls (NSGs, segmentation)  
- Monitoring and alerting to detect abnormal behaviour  
- Alignment with broader identity and governance controls across the environment  

---

## 🚀 Planned Implementation

- Deploy VM Scale Sets with standardised configuration (Cloud-Init)  
- Configure Azure Load Balancer for traffic distribution  
- Implement autoscaling rules based on CPU metrics  
- Configure Azure Monitor alerts and action groups  
- Validate behaviour under simulated load  

---

## 🎯 Purpose

This project is intended to complete the **end-to-end cloud architecture**, covering:

- Identity & Access  
- Data Protection  
- Network Security  
- Compute & Resilience
