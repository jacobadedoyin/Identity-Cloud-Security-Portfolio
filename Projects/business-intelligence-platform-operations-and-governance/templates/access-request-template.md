# Analytics Platform Access Request Template

## Purpose

This template provides a repeatable structure for recording and handling access requests for analytics platforms such as Qlik and Tableau.

The aim is to make access requests easier to validate, action, track, and evidence while avoiding informal or inconsistent user administration.

## Request Information

| Field | Value |
|---|---|
| Request ID | REQ-0001 |
| Request Type | Joiner / Mover / Leaver / Access Change |
| Platform | Qlik / Tableau |
| Business Area | Example: Finance |
| Requested By | Dummy requester |
| Requested For | USER-001 |
| Date Requested | YYYY-MM-DD |
| Required By | YYYY-MM-DD |
| Priority | Low / Medium / High |
| Status | New / In Progress / Completed / Rejected / Escalated |

## Access Requirement

| Field | Value |
|---|---|
| Access Level Requested | Viewer / Editor / Creator / Admin |
| Reporting Requirement | Example: Monthly reporting dashboard access |
| Business Justification | Example: Required to view team performance reports |
| Temporary or Permanent Access | Temporary / Permanent |
| Temporary Access End Date | YYYY-MM-DD / N/A |
| Licence Required | Yes / No |
| Existing Access to Review | Yes / No |

## Validation Checklist

| Check | Completed |
|---|---|
| Request contains enough information to action | Yes / No |
| Business need has been confirmed | Yes / No |
| Correct platform has been identified | Yes / No |
| Requested access level is appropriate | Yes / No |
| Licence availability checked | Yes / No |
| Existing access reviewed where relevant | Yes / No |
| Request does not grant unnecessary elevated access | Yes / No |
| Any temporary access has an end date | Yes / No |

## Action Taken

| Field | Value |
|---|---|
| Access Created | Yes / No |
| Access Updated | Yes / No |
| Access Removed | Yes / No |
| Licence Assigned | Yes / No |
| Licence Recovered | Yes / No |
| Licence Tracker Updated | Yes / No |
| Completion Date | YYYY-MM-DD |
| Completed By | Dummy admin |

## Outcome

| Field | Value |
|---|---|
| Final Status | Completed / Rejected / Escalated / Pending Information |
| Reason for Rejection / Escalation | Add non-sensitive reason if relevant |
| Requester Notified | Yes / No |
| Follow-up Required | Yes / No |
| Follow-up Notes | Add non-sensitive notes only |

## Example Access Request Flow

```text
Access request received
        ↓
Check request details
        ↓
Validate business need
        ↓
Confirm platform and access level
        ↓
Check licence requirement
        ↓
Create, update, or remove access
        ↓
Update licence tracker
        ↓
Confirm outcome
```
