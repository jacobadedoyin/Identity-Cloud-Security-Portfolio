# Analytics Platform JML Checklist Template

## Purpose

This checklist provides a repeatable structure for managing joiner, mover, and leaver activity across analytics platforms such as Qlik and Tableau.

The aim is to make user access administration more consistent, reduce stale access risk, and ensure licence records are updated when access changes are made.

## Request Details

| Field | Value |
|---|---|
| Request ID | REQ-0001 |
| Request Type | Joiner / Mover / Leaver |
| Platform | Qlik / Tableau |
| Business Area | Example: Finance |
| Requested Access Level | Viewer / Editor / Creator / Admin |
| Date Received | YYYY-MM-DD |
| Requested By | Dummy requester |
| Completed By | Dummy admin |
| Completion Date | YYYY-MM-DD |

## Example JML Flow

```text
Request received
        ↓
Identify JML type
        ↓
Validate access requirement
        ↓
Create, update, or remove access
        ↓
Update licence tracker
        ↓
Confirm completion
        ↓
Record evidence
```
