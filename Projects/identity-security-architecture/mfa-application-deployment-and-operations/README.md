# 🔐 MFA Application Deployment and Operations

## Overview

This project demonstrates practical IAM and support experience from an MFA application deployment used to strengthen secure access to Qlik and Tableau data platforms.

The work covered user preparation, OIDC-based authentication transition support, access continuity, issue triage, SOPs, documentation, and post-deployment improvement.

![MFA Application Deployment and Operations](./assets/mfa-application-deployment-and-operations.png)

## Project Scope

| Area | Focus |
|---|---|
| User preparation | Communication, readiness guidance, MFA setup support, and user awareness |
| Technical transition | OIDC-based authentication support and secure access validation |
| Access continuity | Helping reduce disruption during MFA and authentication changes |
| Support operations | MFA setup support, access issue triage, troubleshooting, and escalation |
| Documentation | SOPs, FAQs, user guidance, and repeatable support processes |
| Improvement | Review of recurring issues, support trends, and post-deployment refinements |

## IAM Skills Demonstrated

- MFA adoption and strong authentication support
- OIDC-based authentication transition awareness
- Conditional Access and secure application access awareness
- User onboarding and readiness support
- Access issue triage and escalation
- SOPs, runbooks, FAQs, and operational documentation
- Post-deployment review and continuous improvement

## My Role

I supported the deployment and operational process by helping users prepare for MFA adoption, assisting with MFA setup and access issues, following escalation routes, contributing to support notes, and identifying recurring post-deployment issues.

## Key Outcome

The project strengthened secure access to Qlik and Tableau while supporting users through the transition from preparation to go-live and ongoing operations.

## Confidentiality

This project uses recreated and sanitised evidence only. It does not include real user records, authentication logs, tenant details, tickets, internal screenshots, or confidential rollout material.
