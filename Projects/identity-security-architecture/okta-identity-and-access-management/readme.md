# 🔐 Okta Identity & Access Management

**Authentication, MFA, and Application Access Control using Okta**

---

## 📌 Overview

This project demonstrates practical implementation of identity and access management using Okta, focusing on **authentication, MFA enforcement, and application access control**.

The objective is to simulate a real-world environment where users securely access applications through a centralised identity provider.

---

## 🎯 Objectives

- Configure Okta as an identity provider  
- Enforce Multi-Factor Authentication (MFA)  
- Control access using group-based policies  
- Integrate applications with Okta  
- Validate secure authentication flows  

---

## 🏗️ Architecture Overview

- **Identity Provider:** Okta  
- **Users:** Internal users  
- **Authentication:** MFA  
- **Access Control:** Group-based assignment  
- **Applications:** Integrated test application  

---

## 🔐 Implementation

### 1. User & Group Setup
- Created users in Okta  
- Assigned users to groups  
- Defined access based on group membership  

---

### 2. MFA Enforcement
- Enabled MFA policy  
- Configured authentication factors  
- Applied policy to user groups  

---

### 3. Application Integration
- Integrated application with Okta  
- Configured SSO  
- Assigned access via groups  

---

## 🔍 Validation

- User login required MFA  
- Access restricted based on group membership  
- Unauthorised users blocked from application  

---

## 🧠 Key IAM Concepts Demonstrated

- Centralised identity management  
- Strong authentication (MFA)  
- Role-based access control (RBAC)  
- Secure application access (SSO)
