# Access Governance Summary

## Overview

This evidence summary documents workplace-aligned access governance activity across Qlik and Tableau reporting platforms.

The work focused on supporting controlled user access, joiner / mover / leaver processes, access request handling, and access review activities across analytics environments.

All information in this document is sanitised, recreated, or summarised. No confidential organisational data is included.

---

## Purpose

The purpose of this evidence is to demonstrate practical experience in platform access governance, including:

- Managing user access across reporting platforms
- Supporting joiner, mover, and leaver workflows
- Reviewing access requests against business need
- Supporting least privilege access principles
- Helping maintain visibility of assigned access
- Reducing the risk of stale or inappropriate access
- Documenting access management processes safely

## Access Governance Activities

| Activity | Description |
| :--- | :--- |
| User access administration | Supported the creation, modification, and removal of user access across analytics platforms |
| Joiner support | Helped ensure new users received appropriate access based on business requirements |
| Mover support | Supported access changes when users changed role, team, or reporting requirements |
| Leaver support | Supported access removal to reduce the risk of stale or unnecessary access |
| Access request handling | Reviewed and processed access requests using defined approval or validation steps |
| Access review support | Helped maintain visibility of who had access and whether that access remained appropriate |
| Licence visibility | Supported tracking of assigned licences and identifying unused or recoverable licences |
| Process documentation | Created or improved repeatable guidance for platform access administration |

---

## IAM and Security Relevance

This work is relevant to Identity and Access Management because analytics platforms often contain business-sensitive reporting data.

Controlling access to these platforms supports:

- Least privilege access
- Reduced stale account risk
- Improved joiner / mover / leaver control
- Better visibility of platform access
- Stronger operational governance
- More consistent access request handling
- Improved support for access reviews and audit readiness

## Example Access Governance Workflow

```text
Access request received
        ↓
Business need reviewed
        ↓
Approval or validation confirmed
        ↓
Access provisioned or updated
        ↓
Licence assignment checked
        ↓
Access recorded or tracked
        ↓
Periodic review / removal when no longer required
```
